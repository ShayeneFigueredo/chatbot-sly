O QUE E A MAYA
--------------

A Maya e a atendente virtual da Sly Design no WhatsApp.
Ela responde clientes automaticamente 24/7 usando IA (Groq).

Arquitetura:
  Cliente manda msg no WhatsApp
      ↓
  Baileys (WebSocket) recebe no Render
      ↓
  Encaminha pro Python (backend/webhook_server.py)
      ↓
  Maya processa (maquina de estados + NLP + IA fallback)
      ↓
  Baileys envia resposta de volta
      ↓
  Cliente recebe no WhatsApp

COMO FAZER DEPLOY NO RENDER (MANUAL)
--------------------------------------

O Render faz deploy automatico quando voce da git push no GitHub.
Mas se precisar fazer manual:

1. Entra em https://dashboard.render.com
2. Clica no servico "chatbot-sly-oficial"
3. Clica no botao "Manual Deploy" (canto superior direito)
4. Escolhe "Deploy latest commit"
5. Aguarda o build terminar (~3-5 minutos)

ATENCAO: Cada deploy PERDE a sessao do WhatsApp.
Depois do deploy, voce precisa:
  - Abrir https://chatbot-sly-oficial.onrender.com/qrcode no CELULAR
  - Escanear o QR Code com WhatsApp → Aparelhos Conectados

COMO REEMPARELLHAR (QUANDO O QR APARECER NO RENDER)

2. Abre o navegador do celular
3. Vai em: https://chatbot-sly-oficial.onrender.com/qrcode

ARQUIVOS PRINCIPAIS DO PROJETO
-------------------------------

server.py
  Servidor webhook (versao Meta Cloud API — nao usada agora)

backend/webhook_server.py
  Servidor Python com a Maya (maquina de estados + endpoints)
  Endpoints: GET /health, GET /qrcode, POST /responder

connect_whatsapp.js
  Conexao com WhatsApp via Baileys (WebSocket)
  Recebe msgs, encaminha pro Python, envia resposta

app/config.py
  Configuracoes: SYSTEM_PROMPT, PRECOS, TEMAS_DISPONIVEIS, Groq

app/nlp.py
  Detector de intencoes (preco, prazo, busca de tema, suporte, etc)

app/buscador.py
  Busca temas no catalogo CSV (com tolerancia a erro de digitacao)

backend/.env
  Variaveis de ambiente: GROQ_API_KEY, META_TOKEN
  NAO SOBE PRO GIT (esta no .gitignore)

Dockerfile
  Configuracao do container Docker (Python + Node.js)

start.sh
  Script de inicio do Render (inicia Python + Baileys)

FLUXO DA CONVERSA (MAQUINA DE ESTADOS)
---------------------------------------

A Maya usa maquina de estados. Cada cliente tem seu estado salvo:

  menu → slides_prontos / pedido_tema / buscando_tema / precos / duvida

Fluxo de pedido completo:
  pedido_tema → pedido_tipo → pedido_qual_tema → pedido_prazo
  → pedido_nomes → pedido_extras → resumo → pagamento

Comandos da Shay (dona):
  Quando voce manda msg no WhatsApp da Sly, a Maya reconhece
  e nao chama IA. Comandos:
    assumir NUMERO    — assume atendimento manual
    liberar NUMERO    — devolve pra Maya
    humanos           — lista clientes em atendimento humano



QUANDO ALGO DER ERRADO
-----------------------

1. "Maya nao responde"
   → Verificar se o Render ta online: curl /health
   → Ver logs do Render: https://dashboard.render.com

2. "QR Code aparecendo de novo"
   → Sessao foi perdida (deploy ou crash)
   → Escanear QR em: https://chatbot-sly-oficial.onrender.com/qrcode

3. "Render excedeu memoria"
   → Normalmente e o processo antigo (wppconnect com Chrome)
   → O Baileys atual usa bem menos memoria
   → Se acontecer, fazer deploy manual

4. "Erro 401 no test_whatsapp.py"
   → Token do Meta venceu (token de teste expira rapido)
   → Nao afeta o funcionamento (Baileys nao usa token da Meta)

5. "Maya respondendo coisa errada"
   → Ver o log do Render (mostra a intencao detectada e a resposta)
   → Ajustar palavras-chave em app/nlp.py
   → Ajustar SYSTEM_PROMPT em app/config.py
